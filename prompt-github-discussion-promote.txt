Log the start time immediately and record the current date with `Get-Date -Format o`.

You are running the Bigbsky GitHub discussion-promotion loop from the current repo/worktree. Do not switch to `N:\Projects\bigbsky-github` when this prompt is running from a lane worktree.

Goal:
- Promote one GitHub Discussion labeled `ai:ready-to-promote` into a GitHub Issue.
- Preserve the discussion body and comments in the issue body.
- Label the new issue `ai:needs-roast`. The promote script also carries forward exactly these labels from the discussion when present: `short-term`, `long-term`, and `priority:urgent`. There is no generic "source label" propagation beyond those three.
- Set the issue type automatically: new capability = `Feature`, broken/fixing previous work = `Bug`, planning/workflow/test/spec work = `Task`.
- Mark the discussion `ai:promoted` and leave a neutral workflow comment linking the issue.
- Stop after one promotion.

Rules:
- Do not edit application code.
- Do not edit `changelog/internal/CHANGELOG.md`.
- Do not put secrets into issues, discussions, prompt files, logs, or generated artifacts.
- The script's urgent pre-flight check excludes discussions labeled `ai:claimed`, `ai:blocked`, or `ai:needs-user-answer` when deciding whether an urgent discussion is still pending triage. The main ready-to-promote candidate selection only excludes `ai:claimed` and `ai:promoted`. If you see a candidate that also carries `ai:blocked` or `ai:needs-user-answer`, stop and surface the conflict in a workflow comment rather than promoting it — that combination should not exist.
- If no ready discussion exists, print that no discussion was available and stop.

Process:
1. Dry run the next promotion:
   ```powershell
   .\scripts\github-loop\promote-discussion.ps1
   ```
2. Read the generated issue body path shown by the script.
3. The dry run also prints GitHub's native browser URL for `Create issue from discussion`, for example:
   `https://github.com/radialmonster/bigbsky-dev/issues/new?created_from_discussion_number=N`
   Use that URL for manual browser-based conversion when native GitHub discussion linkage matters more than unattended automation.
4. If it is safe to promote unattended, apply:
   ```powershell
   .\scripts\github-loop\promote-discussion.ps1 -Apply
   ```
5. Verify the created or existing issue and discussion labels/comments with GitHub CLI.
6. Print a concise summary: discussion number, issue URL, native GitHub URL if manual conversion was skipped, and whether any promotion was skipped.

End:
- Log end time and elapsed minutes.
